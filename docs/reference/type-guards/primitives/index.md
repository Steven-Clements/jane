# Overview

Primitive type guards check whether a value is one of JavaScript’s built‑in primitive types. They return simple booleans, never throw, and never mutate input.

- [isBigInteger](is-big-integer.md): Checks whether a value is a `bigint` primitive.
- [isBoolean](is-boolean.md): Checks whether a value is a boolean.
- [isFiniteNumber](is-finite-number.md): Checks whether a value is a finite number.
- [isInteger](is-integer.md): Checks whether a value is an integer.
- [isNonEmptyString](is-non-empty-string.md): Checks whether a value is a non‑empty string.
- [isNonNegativeNumber](is-non-negative-number.md): Checks whether a value is a non‑negative number.
- [isNull](is-null.md): Checks whether a value is `null`.
- [isNullOrUndefined](is-null-or-undefined.md): Checks whether a value is `null` or `undefined`.
- [isNumber](is-number.md): Checks whether a value is a number.
- [isPositiveInteger](is-positive-integer.md): Checks whether a value is a positive integer.
- [isSafeInteger](is-safe-integer.md): Checks whether a value is a safe integer.
- [isString](is-string.md): Checks whether a value is a string.
- [isSymbol](is-symbol.md): Checks whether a value is a symbol.
- [isUndefined](is-undefined.md): Checks whether a value is `undefined`.

All primitive guards are pure, predictable, and safe to use before normalization or validation.
